# 终端音游 屏幕刷新原理

## 为什么不能用 system("cls")

最朴素的终端刷新方式是每帧 `system("cls")` 清屏再重新打印。问题是：

1. `cls` 会清空整个屏幕缓冲区，终端需要重绘所有像素——产生可见的白屏闪烁
2. 清屏和重绘之间有时间差，帧率越高闪烁越明显
3. `system()` 本身要启动一个子进程，开销大（约几毫秒）

## 本项目的方案：光标归位 + 原地覆写 + 单次写入

核心思路：**不清屏，把光标移回左上角，用新内容直接覆盖旧内容。**

### 第一步：备用屏幕缓冲区

```c
printf("\033[?1049h");  // 进入备用屏幕
printf("\033[?1049l");  // 退出备用屏幕
```

终端有两个屏幕缓冲区：主屏幕（你平时看到的）和备用屏幕（全屏程序如 vim、less 用的）。游戏开始时切到备用屏幕，结束时切回来。好处：

- 游戏画面不会污染终端的滚动历史
- 退出后终端恢复到游戏开始前的状态

### 第二步：光标归位（不清屏）

```c
sprintf(buf+len, "\033[H");
```

`\033[H` 是 ANSI 转义序列，意思是"把光标移动到第1行第1列"。这比 `cls` 快得多——不清除任何内容，只是改变下一个字符的写入位置。

### 第三步：逐行覆写 + 行尾清除

```c
sprintf(buf+len, " GOOD:%-4d  BAD:%-4d  MISS:%-4d\033[K\n", good, bad, miss);
```

每行末尾的 `\033[K` 是"清除从光标到行尾"。作用：

- 如果新内容比旧内容短，旧内容的尾部残留会被清掉
- 如果新内容更长或一样长，`\033[K` 什么都不做（无副作用）

这样每行都是"写新内容 → 擦掉可能的残留"，不需要先清整行再写。

### 第四步：帧尾清除

```c
sprintf(buf+len, "\033[J");
```

`\033[J` 是"清除从光标到屏幕底部的所有内容"。放在整帧最后，确保画面下方没有上一帧遗留的多余行。

### 第五步：缓冲区一次性写入

```c
char buf[4096];
int len = 0;

// 所有 sprintf 都写入 buf...

#ifdef _WIN32
WriteConsoleA(hOut, buf, (DWORD)len, &written, NULL);
#else
::write(STDOUT_FILENO, buf, len);
#endif
```

关键：**不用 cout/printf 逐行输出，而是把整帧拼到一个 char 数组里，最后一次系统调用写出去。**

原因：
- `cout` / `printf` 每次调用可能触发一次系统调用或缓冲区 flush
- 多次写入之间如果终端来得及渲染，就会看到"画了一半"的中间状态
- 单次 `WriteConsoleA` / `write` 把整帧作为一个原子操作交给终端，要么全显示要么没显示

### 第六步：精确帧定时

```c
auto next_frame = std::chrono::steady_clock::now();

// 每帧循环末尾：
next_frame += std::chrono::milliseconds(8);
std::this_thread::sleep_until(next_frame);
```

不用 `Sleep(8)`，因为 Sleep 是"至少睡8ms"，加上帧内逻辑时间会越来越慢。

`sleep_until` 是"睡到指定时刻"——每帧的目标时刻是上一帧 +8ms，无论帧内逻辑花了多久（只要不超过8ms），下一帧都准时开始。帧率严格锁定在 125fps。

## 完整流程图

```
游戏开始
  │
  ├─ printf("\033[?1049h")    切到备用屏幕
  ├─ printf("\033[2J\033[H")  清空备用屏幕
  ├─ hide_cursor()            隐藏光标（避免闪烁的方块）
  │
  ╔══════════ 每帧循环 ══════════╗
  ║                              ║
  ║  1. 读键盘输入（非阻塞）     ║
  ║  2. 判定逻辑                 ║
  ║  3. 拼接整帧到 buf[]         ║
  ║     - \033[H 光标归位        ║
  ║     - 逐行写内容 + \033[K   ║
  ║     - \033[J 清除底部残留    ║
  ║  4. WriteConsoleA/write 一次输出  ║
  ║  5. sleep_until(下一帧时刻)  ║
  ║                              ║
  ╚══════════════════════════════╝
  │
  ├─ printf("\033[?1049l")    切回主屏幕
  ├─ show_cursor()            恢复光标
  └─ 显示结算画面

游戏结束
```

## 用到的 ANSI 转义序列速查

| 序列 | 含义 |
|------|------|
| `\033[H` | 光标移到第1行第1列 |
| `\033[K` | 清除光标到行尾 |
| `\033[J` | 清除光标到屏幕底部 |
| `\033[2J` | 清除整个屏幕 |
| `\033[?25l` | 隐藏光标 |
| `\033[?25h` | 显示光标 |
| `\033[?1049h` | 切换到备用屏幕缓冲区 |
| `\033[?1049l` | 切换回主屏幕缓冲区 |

## Windows 兼容性

Windows 的 cmd/终端默认不支持 ANSI 转义序列，需要手动开启：

```c
HANDLE hOut = GetStdHandle(STD_OUTPUT_HANDLE);
DWORD mode = 0;
GetConsoleMode(hOut, &mode);
SetConsoleMode(hOut, mode | ENABLE_VIRTUAL_TERMINAL_PROCESSING);
```

开启后 Windows Terminal / cmd 都能正确解析上述所有转义序列。

## 对比

| 方式 | 闪烁 | 滚动残留 | 帧率上限 | 复杂度 |
|------|------|----------|----------|--------|
| system("cls") + cout | 明显 | 无 | ~30fps | 低 |
| \033[H + cout 逐行 | 轻微 | 可能 | ~60fps | 中 |
| \033[H + buf 一次写 | 无 | 无 | 125fps+ | 中 |
| 备用屏幕 + buf 一次写 | 无 | 无 | 125fps+ | 本项目方案 |
